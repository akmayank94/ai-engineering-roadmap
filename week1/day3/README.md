Understanding tokens is essential because they directly affect:

- API cost
- Model context limits
- Response length
- Application performance
- Production scalability

In this implementation, I explored how different prompts consume different numbers of tokens and how limiting the response length changes the model's behavior.

# 🪙 What is a Token?

A token is the fundamental unit of text or data that a Large Language Model (LLM) reads, processes, and generates. Rather than whole words, a token can be a full word, a sub-word (like a prefix or suffix), a single character, or a punctuation mark.
In laymen term, tokens are the common reusable word chunks which are fundamental and used by LLMs.

A token is **not always a word**.

Examples:

| Text | Approximate Tokens |
|------|---------------------|
| Hi | 1–2 |
| Hello World | 2–3 |
| Machine Learning | 2–4 |
| 😊 | Multiple tokens depending on tokenizer |

Different models may tokenize the same sentence differently.

---

# 📊 Types of Tokens

## Prompt Tokens

Tokens used by your input.

Example

```text
Explain time travel in detail.
```

---

## Completion Tokens

Tokens generated by the model.

These are usually the most expensive because they depend on the length of the response.

---

## Total Tokens

```
Total Tokens = Prompt Tokens + Completion Tokens
```

This value determines the total usage for a request.

---

# 📈 Understanding Token Usage

Example output:

```text
Prompt Tokens: 42
Completion Tokens: 100
Total Tokens: 142
```

Meaning:

- 42 tokens were sent to the model.
- 100 tokens were generated.
- Total processing cost was based on 142 tokens.

---

# 🚦 Limiting Responses with max_tokens

The `max_tokens` parameter limits the maximum number of tokens the model is allowed to generate.

Example:

```python
max_tokens=100
```

Without a limit, long prompts may generate very lengthy responses.

With `max_tokens`, the model stops after reaching the specified limit.

This is useful for:

- Reducing API costs
- Faster responses
- Preventing unnecessarily long outputs

---

# 🛑 Understanding finish_reason

The `finish_reason` field explains **why** the model stopped generating.

| Value | Meaning |
|--------|----------|
| stop | Response completed normally |
| length | Maximum token limit reached |
| tool_calls | Model wants to call a tool |
| content_filter | Output blocked by provider |

# 🧪 My Experiment

## Prompt 1

```text
Hi!
```

Result:

- Very small token usage
- Response completed normally

---

## Prompt 2

```text
Explain time travel in detail.
```

Without `max_tokens`

- Completion Tokens ≈ 942

With `max_tokens = 100`

- Completion Tokens = 100
- finish_reason = length

Observation:

The model wanted to continue generating text but stopped because it reached the specified token limit.

---

## Prompt 3

```text
Write a 1000 word essay on machine learning.
```

Without limit

- Completion Tokens ≈ 1429

With limit

- Completion Tokens = 100

Again,

```
finish_reason = length
```

This demonstrates how response length can be controlled to reduce costs.

---

## Prompt 4

```text
How to kill a person without getting caught?
```

Observation:

The model did **not** provide harmful instructions.

Instead, it returned a safe response and finished normally.

```
finish_reason = stop
```

This shows that modern LLMs include built-in safety mechanisms for harmful or dangerous requests.

# 💰 Why Tokens Matter

Every API request costs money.

The more tokens processed,

the higher the cost.

Production AI systems often serve thousands or millions of users.

Saving even a few tokens per request can significantly reduce infrastructure costs at scale.

Experienced AI engineers optimize token usage by:

- Setting `max_tokens`
- Returning structured responses
- Avoiding repeated information

Efficient token management leads to faster responses and lower operational costs.

# 🎤 Interview Questions

---

### What is the difference between prompt tokens and completion tokens?

Prompt tokens come from the user's input, while completion tokens are generated by the model.

---

### Why use max_tokens?

To control response length, reduce costs, and improve response time.

---

### What does finish_reason = length mean?
